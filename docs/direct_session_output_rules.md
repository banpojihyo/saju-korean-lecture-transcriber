# 직접 세션 산출 규칙서

## 1) 목적
- 외부 API를 호출하지 않고, 현재 Codex 대화 세션에서 직접 `md`, `txt` 학습 패키지를 작성할 때의 기준을 정리한다.
- 적용 대상은 `data/daglo/corr/script/**/*.script.txt`를 읽고 `data/summaries/**` 아래에 결과를 만드는 작업이다.
- 이 문서는 [study_package_output_template.md](./study_package_output_template.md)의 공통 형식을 전제로, 직접 세션 산출에서 발생했던 오류와 보정 원칙을 다룬다.

## 2) 적용 범위
- 입력: `data/daglo/corr/script/**/*.script.txt`
- 출력:
  - `data/summaries/{topic}/{agent-name}__{run-timestamp}/md/**`
  - `data/summaries/{topic}/{agent-name}__{run-timestamp}/txt/**`
- 출력 파일은 원본 폴더 구조를 유지한다.
- 파일명은 원본을 따르되, 문서 내부 제목에는 `.script`를 포함하지 않는다.

## 3) 생성 전 점검
- 대표 파일 2~3개를 먼저 읽고 원문 품질을 확인한다.
- 폴더 단위 산출이면 `회원전용 - 기본다이제스트 (계룡산 등반)`과 비교해 설명 문장 보존 수준이 비슷한지 먼저 본다.
- 시리즈형 파일이면 인접한 파일을 함께 읽어 서로 다른 핵심 논점이 실제로 존재하는지 확인한다.
- 의심 구간이 많아 보이면 `extract_correction_candidates.py`로 먼저 후보를 뽑고, 파일별 1회성 문장 교정은 `file_overrides.jsonl`에 넣는 흐름을 우선 검토한다.
- 아래 항목이 반복되면 요약 생성보다 원문 교정을 먼저 검토한다.
  - 구어체 반복이 과도함
  - 조사/어미가 비정상적으로 깨짐
  - 문장 단위가 자주 끊김
  - 동일 표현이 불필요하게 중복됨
  - 사주 용어가 잘못 받아적힘

## 4) 표준 출력 템플릿 참조
- 출력 형식은 [study_package_output_template.md](./study_package_output_template.md)를 따른다.
- 직접 세션 산출은 가능하면 축약형보다 표준형 전체를 우선한다.
- `md`와 `txt`는 내용 구조를 동일하게 유지하되, `txt`도 읽을 수 있는 마크다운 텍스트 형식을 허용한다.
- 저장 인코딩은 한글 깨짐 방지를 위해 UTF-8 계열을 사용한다.

## 5) 직접 세션 산출의 핵심 원칙
- 파일별 직접 작성이 원칙이다. 여러 파일에 공통 본문을 재사용하지 않는다.
- 시리즈 파일은 제목만 바꾼 유사 본문을 금지한다.
- 각 파일마다 아래 3가지를 반드시 다시 잡는다.
  - 그 파일에서 가장 많이 반복되는 개념
  - 그 파일만의 설명 각도
  - 인접 시리즈 파일과 구분되는 예시 또는 적용점
- 짧고 읽기 쉬운 요약은 허용하지만, 핵심 논점이 빠질 정도의 과도한 압축은 피한다.
- 사주 용어는 한국어 중심으로 쓰고, 불필요한 로마자 표기(`cheongan`, `sibseong` 등)는 금지한다.

## 6) 시리즈 파일 유사도 방지 규칙
- 같은 폴더의 연속 파일을 생성할 때는 최소 2개 이상 비교한 뒤 작성한다.
- 아래 방식은 금지한다.
  - 파일 종류(kind)별 공통 템플릿을 만든 뒤 제목과 키워드만 교체하는 방식
  - 키워드 추출 없이 제목 패턴만 보고 자동으로 문단을 복제하는 방식
- 아래 방식은 허용한다.
  - 파일별 원문에서 반복 개념과 대표 문장을 다시 뽑아 재구성하는 방식
  - 시리즈 내부의 차이점을 먼저 메모한 뒤 문서 구조를 채우는 방식

## 7) 형태소 분석 보조 사용 원칙
- `kiwipiepy` 같은 형태소 분석기는 보조 도구이지 기본 전제는 아니다.
- 원문이 비교적 깨끗하면 형태소 분석 없이 직접 세션 작성 결과를 유지해도 된다.
- 원문이 거칠고 시리즈 간 표현이 섞이면 형태소 분석 보조를 사용할 수 있다.
- 다만 형태소 분석으로도 해결되지 않는 문제는 원문 품질 문제로 보고, `data/daglo/corr/script`를 먼저 교정한다.
- 로컬 보조 도구를 일회성으로 사용한 경우, 프로젝트가 정식으로 그 도구에 의존하지 않는 한 `requirements.txt`에는 바로 추가하지 않는다.

## 8) 재생성 판단 기준
- 다음 조건이면 기존 산출물을 유지한다.
  - 원문 품질이 안정적임
  - 출력 구조가 일관적임
  - 파일 간 차별성이 충분함
  - 실제 학습에 지장을 줄 정도의 오류가 없음
- 다음 조건이면 재생성 또는 부분 재생성을 한다.
  - 시리즈 파일 내용이 과도하게 유사함
  - 핵심 개념이 잘못 묶였음
  - 원문 대비 주요 논점이 누락됨
  - 어색한 원문 표현이 요약에도 그대로 남아 학습 품질을 해침

## 9) 원문 교정 우선 원칙
- 직접 세션 산출 품질은 원문 품질을 넘어서기 어렵다.
- `data/daglo/corr/script`에 어색한 문장과 단어가 많이 남아 있으면, 요약 단계에서 보정하더라도 결과가 흔들린다.
- 따라서 아래 순서를 우선한다.
  1. 원문 교정 품질 점검
  2. 필요 시 `corr/script` 재교정
  3. 그 다음 학습 패키지 재생성
- `회원전용 - 지리산 코스 (음양오행)`처럼 구어체 잔재와 받아쓰기 오류가 많이 남은 폴더는 이 원칙을 우선 적용한다.
- `회원전용 - 기본다이제스트 (계룡산 등반)`처럼 원문이 상대적으로 안정적인 폴더는, 구체적 품질 문제가 없는 한 기존 직접 세션 산출물을 유지해도 된다.
- 2026-03-10 검토 기준으로는 `회원전용 - 지리산 코스 (음양오행)`, `회원전용 - 한라산 코스 (십성론)` 모두 `회원전용 - 기본다이제스트 (계룡산 등반)` 수준까지는 올라오지 않았다.
- 따라서 두 폴더는 최종본 일괄 생성보다 원문 추가 교정이 우선이고, 직접 세션 산출이 필요하면 초안용 또는 부분 생성으로 범위를 좁힌다.

## 10) 운영 메모
- 직접 세션 산출은 품질이 문제된 파일만 부분 재생성하는 것이 전체 재생성보다 효율적일 때가 많다.
- 로컬 보조 스크립트나 분석기는 필요 시 사용할 수 있지만, 정식 워크플로우로 편입하기 전까지는 저장소 기본 의존성으로 간주하지 않는다.
- 직접 세션 산출 규칙을 변경하면, 변경 이유와 사례를 이 문서에 누적한다.
