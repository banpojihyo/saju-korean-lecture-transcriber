# 통합 학습 패키지 출력 템플릿

## 1) 목적
- `data/daglo/corr/script/**/*.script.txt`를 바탕으로 만드는 학습용 요약 산출물의 표준 형식을 정의한다.
- 이 문서는 생성 방식이 API이든 직접 세션이든 공통으로 적용된다.
- 생성 방식별 차이는 운영 규칙에서 다루고, 최종 문서 형식은 가능하면 이 템플릿으로 통일한다.

## 2) 기본 원칙
- 결과는 한국어로 작성한다.
- 문서 내부 제목에는 `.script`를 포함하지 않는다.
- `md`와 `txt`는 내용 구조를 동일하게 유지한다.
- 원문에 없는 사실을 단정하지 않는다.
- 불확실한 내용은 중립적으로 표현한다.
- 자동 초안은 그대로 최종본으로 확정하지 않는다.
- 최종본은 파일별 재작성 기준으로 다듬고, 시리즈 파일 간 차이를 명확히 드러내야 한다.

## 3) 표준 제목 형식
- 제목:
  - `# {원본 제목} 통합 학습 패키지`
- 예:
  - `# 기본 다이제스트 01 - 한난조습이란？ 통합 학습 패키지`

## 4) 표준 섹션 구조
각 출력 파일은 아래 5개 섹션을 기본으로 유지한다.

1. `## 핵심 주제별로 나눠서 정리`
- `### 주제 1`, `### 주제 2` 식으로 핵심 논점을 분리한다.
- 각 주제는 bullet point 중심으로 정리한다.
- 각 주제마다 최소 1개의 파일별 설명 bullet이 들어가야 한다.
- 원문 문장을 그대로 옮기는 bullet만으로 채우지 않는다.
- 시리즈 파일이면 인접 파일과 겹치지 않게 해당 파일만의 관점을 반영한다.

2. `## 핵심 개념 맵`
- 핵심 개념 간 관계를 간단한 bullet으로 정리한다.
- 용어 정의만 나열하지 말고, 개념 간 연결과 작용 방향을 보여준다.
- 최소 1개 이상은 `A -> B` 또는 `A는 B와 연결된다` 수준의 관계 서술을 포함한다.

3. `## 예상 시험문제`
- 최소 5개 문항을 권장한다.
- 각 문항은 질문, 정답, 짧은 해설을 포함한다.
- 단답형, 서술형, 개념 비교형, 적용형을 섞는다.
- 동일한 질문 골격을 여러 파일에 반복하지 않는다.
- `이 파일에서 X가 중요한 이유는 무엇인가?` 같은 고정 질문 틀만 반복하는 방식은 금지한다.

4. `## 시험 포인트와 실전 주의사항`
- 시험에 자주 나올 수 있는 핵심 포인트를 압축한다.
- 헷갈리기 쉬운 개념, 오해하기 쉬운 표현, 실전 해석상 주의점을 함께 적는다.
- 파일별 포인트를 써야 하며, 범용 문구를 그대로 복제하지 않는다.

5. `## 꼭 공부해야 할 내용`
- 최종 복습용 핵심 내용을 정리한다.
- 암기 포인트, 개념 연결, 적용 관점을 우선순위 있게 남긴다.
- 단순 키워드 나열보다 “어떤 순서로 이해해야 하는가”를 드러내는 것이 좋다.

## 5) 생성 방식별 허용 차이
- Direct Session 최종본:
  - 표준형 전체를 유지한다.
  - 파일별 논점 차별화와 문장 품질을 가장 강하게 본다.
  - 문제/포인트/주의사항을 파일별로 다시 쓴다.
- 자동 초안:
  - 상위 5개 섹션을 유지하되, direct session 최종본의 초안으로 취급한다.
  - 핵심 개념 후보, 대표 문장, 파일별 focus seed를 빠르게 모으는 역할에 집중한다.
  - 결과를 그대로 최종본으로 확정하지 않는다.
- API 자동 생성:
  - 상위 5개 섹션은 가능하면 유지한다.
  - 토큰 한도나 길이 제약이 있을 때는 하위 bullet 수와 문제 수를 줄일 수 있다.
  - 단, 상위 섹션을 임의로 3개 블록으로 축소하기보다는 같은 5섹션을 유지한 채 밀도를 조정하는 것을 우선한다.

## 6) 축약형 허용 기준
- 아래 조건에서는 축약형을 허용한다.
  - 원문이 매우 길어 출력 길이 제약이 강함
  - API 호출 비용 또는 토큰 제한 때문에 하위 항목 수를 줄여야 함
  - 중간 단계 요약본을 임시로 저장하는 경우
- 축약형에서도 아래는 유지한다.
  - 제목 형식
  - 상위 5개 섹션 이름
  - 최소한의 시험문제와 복습 포인트
- 단, Direct Session 최종본은 가능하면 축약형 대신 표준형으로 복원한다.

## 7) 권장 방향
- 최종 사용자용 산출물은 `5섹션 표준형`을 기본으로 한다.
- 자동 초안은 `빠른 초벌`, direct session은 `최종 문장 품질 확보`로 역할을 분리한다.
- Direct Session과 API는 형식을 다르게 가져가기보다, 같은 형식을 공유하고 밀도만 다르게 운영하는 편이 비교와 유지보수에 유리하다.
